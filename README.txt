This file contains a breakdown of the github repo structure.
If you're looking for a breakdown of the project, reference 
the project plan in the doc folder.

doc:
	This folder contains the primary artifacts for the 
	project such as the project plan, requirements
	specification, etc.
src:
	This folder contains any source code for the project.
lib:
	This folder contains header files for the project.
test:
	This folder contains test cases and other artifacts
	relating to testing the software.
